◆ fibered_process_node_t

typedef struct fibered_process fibered_process_node_t

A node of the list of processes in fibered_processes_list, a fiber-enabled process.

A fibered_process is a process in which at least one thread has been converted to a fiber, so a fibered_process always has at least one fiber in the fibered_process::fibers_list field. Moreover the first element of the list has fibered_process::prev pointing to NULL and the last element of the list has fibered_process::next pointing to NULL.

◆ fibered_processes_list_t

typedef struct fibered_processes_list fibered_processes_list_t

The list of processes that have at least one fiber.

This list has the purpose of containing the list of processes that have created at least one fiber and they became fiber-enabled. The list can be considered empty when at least one of fibered_processes_list::head or fibered_processes_list::tail is pointing to NULL.

◆ fibers_list_t

typedef struct fibers_list fibers_list_t

A generic list of fibers.

The list has the purpose of containing the list of fibers for a given process. The list can be considered empty when at least one of fibers_list::head or fibers_list::tail is pointing to NULL.

◆ convert_thread_to_fiber()

int convert_thread_to_fiber ( void  )

Convert the current thread to a fiber.

Implementation

When a thread is converted to a fiber several task are performed. First of all, if the process never created a fiber, it must become fiber-enabled, this means that we have to instantiate a fibered_process element in the fibered_processes_list variable. Then we have to instantiate a fiber element in the fibers_list field of the fibered_process element of the list.

Returns
int 0 if everything ok, otherwise an error listed in ioctlcmd.h
